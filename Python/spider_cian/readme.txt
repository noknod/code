
ОС разработки MS Windows XP SP2

(для работы в Windows требуется установка 
 http://sourceforge.net/projects/pywin32/
)


Данные извлекаются с сайта www.cian.ru, при этом категории объектов 
недвижимости возможно задать при вызове scrapy с помощью параметров:

- deal_type - сделка:
    - rent - аренда,
    - sale - продажа,
    - по умолчанию принимается первый тип из кортежа 
      cian_project.const.py.DEAL_TYPES, равный u'rent';

- realty - вид объекта недвижимости:
    - flats - жилой,
    - offices - коммерческий,
    - suburbian - загородный,
    - по умолчанию принимается первый вид из кортежа 
      cian_project.const.py.REALTY_TYPES, равный u'flats';

- region - месторасположение, по умолчанию г. Екатеринбург
  (START_URI_PART_REGION = u'obl_id=43&raion_obl_id[1]=347')
;

- params - дополнительные параметры отбора в том же виде, в каком сайт 
осуществляет поиск (например, 'wp=1&room1=1' означает 1-комнатные квартиры и
объявления только с фото).


В файле cian_project.const.py задаются минимальные значения для просмотра 

- GRAB_CNT_PAGE_MIN - количества страниц, по умолчанию 2;

- GRAB_CNT_ITEM_MIN - количество ссылок, по умолчанию 20.

При этом количество страниц имеет приоритет над количеством ссылок (то есть, 
пока не будет просмотрено минимальное количество страниц, поиск будет 
продолжаться, даже если ссылок уже просмотрено достаточно).


В файле cian_project.const.py константой IS_NEED_JSON (по умолчанию True) 
задаётся необходимость сохранения результата через pipeline в файл формата 
json (который будет создан в каталоге проекта), при этом происходит 
кодирование из Unicode в UTF-8.
