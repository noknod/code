Задание:
Консольная утилита для скачивания файлов по HTTP протоколу.

- - -

Операционная система разработки и тестирования
Windows XP SP 2

Версия Java
java version "1.6.0_21"
Java(TM) SE Runtime Environment (build 1.6.0_21-b07)
Java HotSpot(TM) Client VM (build 17.0-b17, mixed mode, sharing)

- - -

Структура файлов и папок в решении

http_downloader
	|
	-	out		папка для тестовых выходных файлов
		|
		-	(out.txt)		во время работы примера лог записывается в этот файл
		-	(err.txt)		во время работы примера учтённые ошибки записывается сюда
		-	(*.bmp/jpg)		скачанные файлы во время работы примера

	-	prog		папка с исходными кодами и файлом для создания исполняемого архива
		|
		-	src			папка с исходными кодами
		-	build.xml		файл для создания исполняемого архива посредством ant
		-	(build\classes)	после компиляции ant создаст эту папку с байт-кодами классов
		-	(dist\lib)		созданные ant`ом библиотеки будут помещены в эту папку
		-	(dist\kyatest.jar)	итоговый исполянемый архив-утилита

	-	make.bat	make-файл, использующий ant для сборки

	-	run.bat	bat-файл с примером работы, использует test.txt как источник ссылок

	-	test.txt	файл с примерами ссылок для скачивания

	-	readme.txt	соответственно данный файл со справкой

- - -

Итоговый исполняемый архив, решающий задачу, будет создан ant`ом в папке prog\dist\ под именем download.jar.

На вход принимаются параметры

	-n  количество одновременно качающих потоков (1,2,3,4....)
	-l   общее ограничение на скорость скачивания, для всех потоков, размерность - байт/секунда, 
		можно использовать суффиксы k,m (k=1024, m=1024*1024)
	-f   путь к файлу со списком ссылок
	-o  имя папки, куда складывать скачанные файлы
	-ml файл для записи лога во время работы, если равен sys, то лог будет записываться в стандартный поток 
 		вывода сообщений, при отсутствии параметра логирование вестись не будет
	-me файл для записи учтённых ошибок во время работы, если равен sys, то лог будет записываться в стандартный 
		поток вывода ошибок, отсутствие параметра равноценно заданию его значением sys

Формат файла со ссылками:

	<HTTP ссылка><пробел><имя файла, под которым его надо сохранить>

По условию, в HTTP ссылке нет пробелов, нет encoded символов - это всегда обычные ссылки с английскими символами без специальных символов.
Ссылки считаются без небходимости авторизации. Также поддерживается только HTTP-протокол.

По условию, ссылки могут повторяться в файле, но с разными именами для сохранения. В этом случае программа копирует уже скачанный файл с 
другим именем. Считается, что имена файлов уникальны.
Было принято решение отказаться от стандартной схемы, когда один (несколько) потоков заполняют очередь, и остальные читают задания.
То есть сперва реализован своего рода map-reduce для ссылок, в итоге получается на каждое задание одно скачивание и, возможно, несколько
копирований.

В конце работы программа выводит статистику - время работы, количество скачанных байт, число скопированных файлов и количество ошибок 
при скачивании и копировании.
